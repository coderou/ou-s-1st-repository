1.谈一谈null和undefined
  1.特点介绍
    -undefined:只有一个值,一般不会主动设置undefined,当出现错误的时候,才会被我们打印出来
    -null:只有一个值,表示一个空的对象指针,这也是typeof null 结果为object的原因
  2.undefined的出现场景:
    -变量被声明但是未赋值
    -函数的形参没有实参传入
    -对象没有赋值的属性
    -函数返回默认值
  3.null的出现场景
    -作为函数的参数,表示该函数的参数不是对象
    -原型链的终点
    -定义变量准备保存对象,最好将变量初始化为null而不是其他值
    -将一个对象变为垃圾对象回收,设置变量值为null

2.什么是DOM
  -Document OBject Model 文档对象模型
  -DOM规定了一系列标准的接口，允许开发人员通过标准的方式，访问、修改、控制节点的样式或行为或内容
  -接口就是函数或者方法,通过接口可以对元素进行操作

3.什么是节点?节点有哪些类型?
  -网页中所有对象和内容都被称作节点:比如元素,注释,文本,属性等等
  -Node(节点)是html中最基本的单元,分为很多类型,所有节点构成DOM树
  -文档节点,元素节点,文本节点,属性节点,注释节点     文档碎片节点
  
4.说出获取元素的所有的方式
  -通过便签名
  -通过id
  -通过类名
  -selectors API

5.新旧获取标签的区别
  -使用旧版本获取元素是动态的(只要元素发生变化,获取的内容就变化)
  -新版本获取元素是静态的,一旦获取完成,则不再受元素修改后的任何影响
